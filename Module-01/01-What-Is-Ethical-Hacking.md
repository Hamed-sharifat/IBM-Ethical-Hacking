# What Is Ethical Hacking?

## Overview

Ethical hacking is the practice of testing systems, networks, and applications to identify vulnerabilities before malicious attackers can exploit them.

Ethical hackers operate with authorization and within legal boundaries to improve an organization's security posture.

---

## Ethical Hacking vs Malicious Hacking

| Ethical Hacker         | Malicious Hacker                  |
| ---------------------- | --------------------------------- |
| Authorized             | Unauthorized                      |
| Legal activities       | Illegal activities                |
| Improves security      | Exploits vulnerabilities          |
| Protects systems       | Damages systems                   |
| Supports organizations | Seeks personal gain or disruption |

---

## Why Ethical Hacking Matters

Organizations face increasing cyber threats that can:

* Cause financial losses
* Disrupt business operations
* Expose sensitive information
* Damage reputation

Ethical hacking helps identify vulnerabilities before attackers can exploit them.

---

## The Five Phases of Ethical Hacking

### 1. Reconnaissance

Gather information about the target.

Example:

* WHOIS queries
* Public information gathering

---

### 2. Scanning and Enumeration

Identify:

* Open ports
* Running services
* Potential vulnerabilities

Example:

* Nmap scans

---

### 3. Exploitation

Safely attempt to exploit vulnerabilities in a controlled environment.

Example:

* SQL injection testing

---

### 4. Post-Exploitation Analysis

Determine what an attacker could accomplish after gaining access.

Example:

* Privilege escalation assessment

---

### 5. Reporting and Remediation

Document findings and recommend security improvements.

Examples:

* Firewall changes
* Patch recommendations
* Security controls

---

## Real-World Example: Equifax Breach

Attackers exploited a known vulnerability that had not been patched.

As a result:

* More than 147 million individuals were affected.
* Sensitive personal information was exposed.

This breach demonstrates the importance of vulnerability assessments and timely patch management.

---

## Key Takeaways

* Ethical hackers think like attackers.
* Ethical hacking requires authorization.
* Ethical hacking is legal and defensive.
* The ethical hacking lifecycle consists of five phases.
* Proactive security testing can prevent major breaches.
